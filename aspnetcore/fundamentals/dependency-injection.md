<span data-ttu-id="136b2-137">L’injection de constructeurs exige qu’un seul constructeur applicable existe.</span><span class="sxs-lookup"><span data-stu-id="136b2-137">Constructor injection requires that only one applicable constructor exist.</span></span> <span data-ttu-id="136b2-138">Les surcharges de constructeurs sont prises en charge, mais une seule peut exister dont les arguments peuvent tous être satisfaits par l’injection de dépendances.</span><span class="sxs-lookup"><span data-stu-id="136b2-138">Constructor overloads are supported, but only one overload can exist whose arguments can all be fulfilled by dependency injection.</span></span> <span data-ttu-id="136b2-139">S’il en existe plusieurs, votre application lève une `InvalidOperationException` :</span><span class="sxs-lookup"><span data-stu-id="136b2-139">If more than one exists, your app will throw an `InvalidOperationException`:</span></span>

### <a name="disposing-of-services"></a><span data-ttu-id="136b2-267">Mise à disposition des services</span><span class="sxs-lookup"><span data-stu-id="136b2-267">Disposing of services</span></span>

<span data-ttu-id="136b2-268">Le conteneur appelle `Dispose` pour les types `IDisposable` qu’il crée.</span><span class="sxs-lookup"><span data-stu-id="136b2-268">The container will call `Dispose` for `IDisposable` types it creates.</span></span> <span data-ttu-id="136b2-269">Toutefois, si vous ajoutez une instance au conteneur vous-même, elle n’est pas mise à disposition.</span><span class="sxs-lookup"><span data-stu-id="136b2-269">However, if you add an instance to the container yourself, it will not be disposed.</span></span>

<span data-ttu-id="136b2-270">Exemple :</span><span class="sxs-lookup"><span data-stu-id="136b2-270">Example:</span></span>

```csharp
// Services implement IDisposable:
public class Service1 : IDisposable {}
public class Service2 : IDisposable {}
public class Service3 : IDisposable {}

public interface ISomeService {}
public class SomeServiceImplementation : ISomeService, IDisposable {}


public void ConfigureServices(IServiceCollection services)
{
    // container will create the instance(s) of these types and will dispose them
    services.AddScoped<Service1>();
    services.AddSingleton<Service2>();
    services.AddSingleton<ISomeService>(sp => new SomeServiceImplementation());

    // container didn't create instance so it will NOT dispose it
    services.AddSingleton<Service3>(new Service3());
    services.AddSingleton(new Service3());
}
```

> [!NOTE]
> <span data-ttu-id="136b2-271">Dans la version 1.0, le conteneur appelé dispose de *tous* les objets `IDisposable`, y compris de ceux qu’il n’a pas créés.</span><span class="sxs-lookup"><span data-stu-id="136b2-271">In version 1.0, the container called dispose on *all* `IDisposable` objects, including those it didn't create.</span></span>
